Project 1:
Technology Trend Analysis

March 2024

Project Goal: Discover what programming languages, database skills, and web frames are the most in demand using data from job postings and surveys.

Introduction
This case study resembles the work a Junior Data Analyst would do for a global IT and business consulting services firm that is known for its expertise in IT solutions. To keep pace with changing technologies and remain competitive, this organization regularly analyzes data to help identify future skill requirements. I determined these future skill requirments using the six data analysis steps: Ask, Prepare, Process, Analyze, Vizualize, and Act. The Jupyter Notebook files I created this analysis can be found here .

Ask
What programming languages, database skills, and Web Frames are the most popular now and are likely to be popular to the future? Properly understanding technology trends is vital in the digital age because it has historically given companies a competetive advantage.

Prepare
I used Beautiful Soup web scraping to find average annual salary by technology, and a web API to find number of jobs by location and technology. After importing this data to a pandas dataframe, I used Openpyxl to convert the dataframes to Excel files. These Excel files were used to create the Number of Jobs by City, Number of Jobs by Technology, and Average Annual Salary by Technology bar graphs in the share section. Finally I created a data frame from a Stackoverflow developer survey, that will be used in the analyze section.

Process
I started cleaning the Stackoverflow data by deleting duplicate rows using the drop_duplicates function. I found missing values in the "Work Location" column using np.nan and replaced them with the most frequent value in the column. Finally, the compensation column was inconsistant because it contained weekly, monthly, and annual compensation. I fixed this problem by creating a "normalized annual compensation" column that multiplyed weekly compensation by 50, monthly compensation by 12, and kept annual compensation the same.

Analyze
After cleaning the data, I used histograms and box plots to understand the distribution of annual salary and age for survey respondents.



The minimum annual salary is $0, lower quartile is $26,169, median is $57,000, upper quartile is $95,000, upper bound is $200,000 and maximum is $2,000,000

I started making visualizations by creating bar charts from the Excel files created with Openpyxl.


After generating the three bar graphs in Excel, I created a dashboard in Cognos Analytics with three tabs: Current Technology Usage, Future Technology Trend, and Survey Demographics. The link to this dashboard can be found here.

Act
The top four findings based on my analysis:

  1. JavaScript is the leading programming language, but Python is close to surpasing JavaScript. TypeScript is also fast growing, and could potentially convert JavaScript users.
  2. MySQL is the most popular database, and PostSQL and MongoDB are gaining popularity. Microsoft SQL Server and SQLite should be avoided due to decreasing popularity.
  3. JQuery, React.js, and Angular/Angular.js are the most popular webframes, but JQuery's growth seems to be stagnating.
  4. Linux and Windows are the most popular platforms, and Docker is quickly approaching them in popularity.

Project 2:
Cyclistic Case Study

August 2022

Project Goal: To determine how annual members and casual riders use Cyclistic bikes differently

Introduction: This case study is for the Google Data Analytics Capstone course and is based on data from Cyclistic, a Chicago based bikeshare company. The study follows the six data analysis steps: Ask, Prepare, Process, Analyze, Visualize, Act. The raw data can be found here and the post-analysis excel spreasheet can be found here.

Ask
Cyclistic has casual riders that pay a small fee for each ride and members who pay an annual fee for unlimited rides. The question for this analysis is how do annual members and casual riders use Cyclistic bikes differently? This answer to this question can be used to encourage more casual riders to switch to annual members, ultimately generating more revenue.

Prepare
The data to be analyzed is from .csv file that contains data for Cyclistic trips for January 2022. This is a first party source since it comes from directly from Cyclistic. It is a comprehensive list all 103,770 trips for the month of January. To prepare the data for analysis, I converted the file to a .xlsx file.

Process
I created column E, "ride_length", by subtracting the start time from the end time and changing to the time format. I also created column F, "day_of_week", using the TEXT function on the "started_at" column. I cleaned the data by using a filter that only just includes rows with rides lasting zero or less minutes and deleting them.

Act
The top three recommendations based on my analysis:

  1. Because memberships are more popular on weekdays than weekends, advertise memberships as a less expensive way to get to work than driving.
  2. Offer a cheaper membership that only allows the customer to ride on the weekends. The weekend membership would target the relatively large amount of casual riders that ride on weekends.
  3. Instead of charging one fee per ride, charge casual rides by a smaller fee per minute of riding. This time based fee insentivises people to get a membership if they would like to ride for a longer time without penalty.
